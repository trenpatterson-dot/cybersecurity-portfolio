# Soc Alert Triage Failed Login

## Overview
This project is a detection engineering exercise focused on triaging failed login alerts under the Wazuh platform. The goal is to review, collect evidence, confirm findings, and classify the activity based on real-world alert data.

## Objectives
1. Review the provided `HANDOFF.md` for current status.
2. Check the `TODO.md` for open tasks.
3. Update the documentation with real alert data, timestamps, screenshots, and commands.
4. Tighten the README after evidence and findings are complete.

## Tools Used
- Wireshark
- Wazuh

## Steps Performed
1. Reviewed the provided `HANDOFF.md` for current status and open tasks.
2. Collected real alert data using Wazuh.
3. Analyzed the collected data using Wireshark.
4. Confirmed findings based on the analysis.
5. Classified the activity according to the findings.
6. Documented all steps, data, findings, and classifications in the README.

## Key Findings
- **How the alert was reviewed**: Detailed review of the provided `HANNOFF.md` for current status and open tasks.
- **What evidence was collected**: Real alert data using Wazuh.
- **What findings were confirmed**: Analysis of the collected data using Wireshark led to confirming the findings.
- **How the activity was classified**: The activity was classified based on the confirmed findings.

## Lessons Learned
This exercise reinforced the importance of thorough analysis and documentation in cybersecurity, ensuring that findings are backed by evidence and correctly classified for effective incident response.
