# SOC Alert Triage Lab

## Overview
SOC Alert Triage Lab is a cybersecurity lab exercise documented under Wazuh, designed to simulate real-world Security Operations Center (SOC) scenarios. The goal was to perform SOC-style alert triage using Wazuh's Security Information and Event Management (SIEM) system.

## Objectives
The primary objective of this lab was to investigate and analyze a series of simulated security alerts generated by repeated failed SSH login attempts from an attacker machine running Kali Linux to a Ubuntu Wazuh server.

## Tools Used
- Wazuh: A open-source, lightweight, and modular SIEM solution for threat detection and response.

## Steps Performed
1. Fixed Wazuh API connectivity issue
2. Verified Wazuh manager and dashboard functionality
3. Configured SSH on the Ubuntu target host
4. Fixed VM networking (host-only configuration)

## Key Findings
- 9 failed SSH login attempts detected from IP: `192.168.32.128` to host `tren` (Ubuntu) using username `fakeuser`. No successful login was observed. [Refer to screenshot: `screenshots/02-failed-login-raw-event.png`]
- Repeated failed logins were filtered and analyzed for further investigation. [Refer to screenshot: `screenshots/03-repeated-failed-logins-filtered.png`]
- Key alert fields such as source IP, user, host, and time were examined for detailed analysis. [Refer to screenshot: `screenshots/04-alert-fields-source-user-host-time.png`]
