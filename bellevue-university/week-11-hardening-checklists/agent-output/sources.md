# Sources

[Lynis](https://cisofy.com/lynis/) — Security auditing and system hardening tool used for the Ubuntu baseline and follow-up scans.

[OpenSSH](https://www.openssh.com/) — SSH service hardened through sshd_config configuration changes.

[unattended-upgrades](https://packages.ubuntu.com/unattended-upgrades) — Ubuntu package used to enable automatic security patching.

[libpam-pwquality](https://packages.ubuntu.com/libpam-pwquality) — PAM module used to enforce password complexity requirements via pwquality.conf.

[VMware Workstation Pro](https://www.vmware.com/products/workstation-pro.html) — Hypervisor used to run the Ubuntu lab VM. Windows Server 2022 VM was not available in this environment.

[CIS Benchmarks](https://www.cisecurity.org/cis-benchmarks) — Reference for Linux and Windows hardening controls and best practices.

[CompTIA Security+ (SY0-701)](https://www.comptia.org/certifications/security) — Week 11 lab scope, Ubuntu hardening tasks, and Windows Server 2022 hardening tasks.
