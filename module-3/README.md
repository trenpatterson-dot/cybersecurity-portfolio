# Module 3 – Threat Hunting & Log Analysis
### CYBR 445 | Bellevue University

---

## 🎯 Lab Objectives

- Perform proactive threat hunting using log data
- Use Linux CLI tools to parse and analyze logs
- Identify Indicators of Compromise (IOCs) in log files
- Understand attacker TTPs mapped to the MITRE ATT&CK framework

---

## 🧠 Key Concepts Learned

- **Threat hunting** — proactive vs reactive detection
- **Log parsing** with grep, awk, sed, cut
- **IOC identification** — suspicious IPs, user agents, file hashes
- **MITRE ATT&CK** framework mapping
- **Pattern recognition** in large log datasets

---

## 🔑 Key Takeaways

1. Threat hunting assumes breach — look for what alerts missed
2. CLI tools like grep and awk are powerful for rapid log triage
3. Knowing attacker TTPs helps focus hunting hypotheses
4. Correlating multiple log sources reveals the full attack chain

---

## 🛠️ Tools Used

- Linux CLI (grep, awk, sed, cut, sort, uniq)
- Wazuh
- MITRE ATT&CK Navigator
