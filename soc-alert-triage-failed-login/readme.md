# Soc Alert Triage Failed Login

## Overview
This project is a detection engineering exercise focused on triaging failed login alerts under Wazuh. The goal was to review, collect evidence, confirm findings, and classify the activity.

## Objectives
1. Analyze failed login alerts for potential security threats.
2. Collect relevant data, timestamps, screenshots, and commands for each alert.
3. Confirm findings and classify the activity based on the evidence collected.
4. Document the process and findings in a clear and concise manner.

## Tools Used
- Wireshark
- Wazuh

## Steps Performed
1. Reviewed the current status from `HANDOFF.md`.
2. Checked open tasks in `TODO.md`.
3. Updated the docs with real alert data, timestamps, screenshots, and commands.
4. Tightened the README after the evidence and findings were complete.

## Key Findings
- Detailed review of the failed login alerts.
- Collection of relevant data, timestamps, screenshots, and commands for each alert.
- Confirmation of findings and classification of activities based on the evidence collected.

## Lessons Learned
Through this exercise, I gained valuable experience in analyzing and triaging security alerts using Wazuh. This project reinforced the importance of thorough documentation and clear communication when dealing with potential security threats.
