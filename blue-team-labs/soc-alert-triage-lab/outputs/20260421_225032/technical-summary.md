## Overview
The SOC Alert Triage Lab is a cybersecurity lab exercise conducted using the Wazuh platform. The goal of this lab was to simulate a real-world scenario where repeated failed SSH login attempts were made from Kali Linux to an Ubuntu Wazuh server, and perform SOC-style alert triage using Wazuh's Security Information and Event Management (SIEM) system.

## Tools & Environment
The environment for this lab consisted of a Wazuh manager running on an Ubuntu machine and an attacker machine running Kali Linux. The Wazuh manager is responsible for collecting, analyzing, and correlating security-related events from the monitored systems.

## Steps Performed
1. Fixed Wazuh API connectivity issue: Ensured that the Wazuh manager could communicate with its API to retrieve and analyze data effectively.
2. Verified Wazuh manager and dashboard functionality: Confirmed that the Wazuh manager and its web-based dashboard were functioning correctly.
3. Configured SSH on Ubuntu target: Set up Secure Shell (SSH) on the Ubuntu machine to allow for secure network connections, including the simulated login attempts.
4. Fixed VM networking (host-only configuration): Adjusted the virtual machine's networking settings to a host-only configuration, isolating it from other networks and ensuring that only traffic within the lab could be observed.
5. Generated failed SSH login attempts from Kali Linux: Simulated repeated failed SSH login attempts from the attacker machine (Kali Linux) to the target Ubuntu server.
6. Captured Wazuh alerts for authentication failures: Utilized Wazuh's alerting system to capture and analyze the security events generated by the repeated failed SSH login attempts.

## Findings
- 9 failed SSH login attempts were detected, originating from IP address 192.168.32.128 (Kali Linux) and targeting the Ubuntu machine with hostname tren.
- The username 'fakeuser' was attempted in all 9 failed login attempts.
- No successful login was observed during this lab exercise.
- Rule ID 2502, with a severity of 10, was triggered by the repeated authentication failures. This rule corresponds to brute force attacks (MITRE ATT&CK T1110).

## Security Significance
The findings from this lab exercise demonstrate the effectiveness of Wazuh's SIEM system in detecting and alerting on potential security threats, such as repeated failed login attempts that may indicate a brute force attack. This lab serves as an example of how to perform SOC-style alert triage using Wazuh, which is valuable for improving incident response capabilities and overall cybersecurity posture.
