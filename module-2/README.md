# Module 2 – Incident Detection & SIEM Fundamentals
### CYBR 445 | Bellevue University

---

## 🎯 Lab Objectives

- Understand SIEM architecture and log ingestion
- Configure and use Wazuh for alert generation
- Analyze security events in Splunk and Elastic Stack
- Practice basic incident triage workflows

---

## 🧠 Key Concepts Learned

- **SIEM fundamentals** — log collection, normalization, correlation
- **Wazuh** — agent deployment, rule configuration, alert management
- **Splunk** — SPL queries, dashboards, alert creation
- **Elastic Stack** — Kibana dashboards, index patterns, KQL queries
- **Incident triage** — severity classification, false positive reduction

---

## 🔑 Key Takeaways

1. SIEM tools are only as good as the rules and tuning behind them
2. Log normalization is critical for cross-source correlation
3. Alert fatigue is a real challenge — tuning reduces noise
4. Understanding normal baselines makes anomalies stand out

---

## 🛠️ Tools Used

- Wazuh SIEM
- Splunk
- Elastic Stack (ELK)
- Linux CLI
